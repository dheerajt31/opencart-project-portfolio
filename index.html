🚀 OpenCart Automation Framework

📌 Project Name:

OpenCart Automation Suite

🏢 Client/Owner:

Self-Initiated by Dheeraj Tayade (for Skill Demonstration)

This is a professional-grade Hybrid Automation Framework built for the OpenCart application, showcasing robust design patterns, reusable components, and continuous integration using Jenkins.

🛠 Tools & Technologies

  • Eclipse IDE
  • Java (Selenium WebDriver Scripts)
  • TestNG Framework
  • Maven for Build Management
  • Page Object Model (POM)
  • Extent Reports for Reporting
  • Log4j for Logging
  • Jenkins for CI/CD Integration
  • GitHub for Version Control

📁 Framework Structure

Well-organized framework with the following layers:

[TestNG Suite (testng.xml)] ↓ [Test Layer (Test Classes)] ↓ [Base Layer (BaseTest.java)] ↓ [Config Layer] ├── config.properties └── ConfigReader.java ↓ [Driver Manager] └── DriverManager.java ↓ [Page Object Layer] └── Page classes (LoginPage, HomePage, etc.) ↓ [Utilities Layer] ├── ExcelUtils ├── WaitUtils └── ScreenshotUtils ↓ [Reporting Layer] ├── ExtentReports └── Log4j ↓ [CI/CD Integration] ├── Jenkins └── GitHub Actions

🔁 Jenkins CI/CD Integration

Jenkins pipeline automatically builds and runs tests on push. Final build status and reports are emailed to team members.

  • ✅ Jenkins Build Report
  • 📨 Automated Email with Extent Report and Summary
